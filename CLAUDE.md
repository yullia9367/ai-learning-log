# CLAUDE.md — 이 저장소의 운영 규칙

이 저장소는 나의 학습을 지식으로 축적하는 LLM wiki다.
LLM 에이전트는 이 저장소에서 작업할 때 반드시 아래 규칙을 따른다.

## 1. 구조 (세 개의 층 + 이 규칙서)

- `2026/` : **raw 층.** 날짜별 학습 기록과 원본 자료(보고서, 대화 발췌, 메모).
  - 절대 수정·삭제하지 않는다. 진실의 원천이다. LLM은 읽기만 한다.
  - 파일명: `2026/MM/DD.md` (하루 기록), `2026/MM/DD-제목.md` (그날의 산출물·자료)
- `wiki/` : **위키 층.** 주제별 지식 페이지. LLM이 쓰고 유지하며 사람이 검토한다.
  - 파일명은 주제 단위: `llm-wiki.md`, `python.md`, `git.md`
- `book/` : **원고 층.** 위키가 충분히 자란 뒤 뽑아내는 산출물. 당분간 비워둔다.
- `CLAUDE.md` : 이 파일. 층이 아니라 전체를 감싸는 규칙서다.

## 2. 위키 페이지 형식

모든 위키 페이지의 항목은 다음 형식을 따른다.

```
## 항목 제목

핵심 내용 한 문장.

- 세부 사항 (필요한 만큼만)

(출처: 2026/MM/DD.md)
```

- 출처는 반드시 raw 파일 링크로 남긴다. 위키에서 raw로 되짚어갈 수 있어야 한다.
- 페이지는 언제 열어도 맥락 없이 읽혀야 한다. 대화 흔적("~인데요", "지난번에")을 남기지 않는다.
- 관련된 다른 위키 페이지가 있으면 마크다운 링크로 연결한다.

## 3. ingest 규칙 (raw → wiki 승격)

- 승격 기준: **한 달 뒤 다시 찾아볼 지식인가?** 그렇다면 승격한다.
- 승격하지 않는 것: 그날의 감상, 일회성 사건, 반복되지 않을 시행착오.
- 새 지식이 기존 페이지와 관련되면 해당 페이지를 **갱신**한다. 새 페이지 남발보다 기존 페이지 갱신을 우선한다.
- 새 정보가 기존 내용과 모순되면 **지우지 말고 병기**하고 `⚠️ 모순` 표시를 남긴다. 어느 쪽이 맞는지는 사람이 판단한다.

## 4. lint 규칙 (점검)

사람이 요청하면 다음을 점검하고 보고한다.

- 위키 안의 중복·모순 내용
- 깨진 링크 (없는 raw 파일, 없는 위키 페이지를 가리키는 링크)
- raw에는 있는데 위키에 승격되지 않은 채 한 달 이상 지난 유의미한 지식

## 5. 금지 사항

- `2026/` 안의 파일을 수정·삭제하는 것
- 사람의 검토 없이 위키를 대량으로 재작성하는 것
- 출처 없는 항목을 위키에 추가하는 것

## 6. 운영 리듬 (사람의 몫 — 참고용)

- 매일: 2026/ 날짜 파일에 raw 기록 5분
- 수시: 좋은 답변을 위키에 저장 (LLM에게 다듬기를 시키고 사람이 검토)
- 월말: ingest + lint 30분
- 매 작업 후: git commit + push
